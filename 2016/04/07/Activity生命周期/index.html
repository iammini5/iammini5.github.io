Activity生命周期

发表于 2016-04-07   |   分类于 Android   |     |   本文总阅读量 次

对于Activity来说,本质上就是以下四种状态:

  1. Activity处于前台,此时它保持活跃的状态.
  2. Activity失去焦点但任然可见(即被透明的或不是全屏的Activity挡住焦点),此时它会暂停(pause),暂停了的Activity任然是alive,它会维护住它的全部变量和状态,但可以在系统内存不足时被kill掉.
  3. Activity完全被其他Activity挡住,那么他会停止(stop),它仍然会维护住它的全部变量和状态,但它会更容易在系统内存不足时被kill掉
  4. 如果Activity被暂停或停止了,系统可以随时kill掉这个Activity或者直接kill整个进程.当它重新出现时,所有的状态必须重新创建.

看一下官方给的一个说明图：

这个图是给了一些基本的方法，下面再逐个说一下。

  1. onCreate(Bundle savedInstanceState)
    表示activity正在被创建，这是生命周期的第一个方法，在这里进行常见的初始化操作，比如绑定UI，初始化控件等。savedInstanceState参数存储了一些在onSaveInstanceState(Bundle)存储的数据，如果没有设置则为null。
  2. onRestart()
    表示activity重新启动，一般情况下，当当前activity从不可见重新变为可见状态时，onRestart()被调用。
  3. onStart()
    在onCreate(Bundle savedInstanceState)或onRestart()后调用，表示activity正在启动，这时的activity已经可见了，但是还没有出现在前台，所以无法与用户进行交互。
  4. onResume ()
    准备与用户进行交互，但这不是最佳的方式去判断当前界面是否对用户可用，但像键盘等系统控件弹出时也会影响焦点，所以最佳方式是使用onWindowFocusChanged (boolean hasFocus))来判断当前界面是否获得了焦点。
  5. onPause ()
    表示activity正在停止，正常情况下，紧接着onStop()会被调用。在此状态下可以做一些存储数据，停止动画等工作，但注意不好太耗时，因为这会影响到新的activity的显示，onPause()必须先执行完，新的activity的onResume()才会执行。当Activity失去焦点但不被kill的时候，onPause()会被调用。只有onPause结束，新Activity的onCreate方法才会开始。
  6. onStop ()
    当Activity对用户已经不可见时调用，但当系统内存较低时，Activity可能在onPause()后没有足够内存继续运行而导致onStop()不被调用，在此状态下可以做一些稍微重量级的回收工作，同样不能太耗时。
  7. onDestroy ()
    在Activity销毁前执行最后工作，主要执行资源释放，内存释放相关操作。
    整个生命周期从onCreate到onDestroy,所以对于一些全局的变量,需要在onCreate初始化,在onDestroy中销毁,例如handler。

可见的生命周期是从onStart到onStop,在这两个方法期间可以维护一些需要改变Activity展示的数据,比如BroadcastReceiver的注册和销毁.
前台的生命周期是onResume到onPause,此时用户可以与Activity交互.
